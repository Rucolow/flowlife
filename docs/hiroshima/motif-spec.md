# Motif Spec — プロシージャルASCIIモチーフ仕様

## 共通仕様

### モチーフ関数のインターフェース

```typescript
type MotifFunction = (
  nx: number,   // 正規化X座標 (0-1)
  ny: number,   // 正規化Y座標 (0-1)
  time: number,  // 経過時間（秒）
  mx: number,   // マウスX (0-1, ヒーローのみ使用)
  my: number,   // マウスY (0-1, ヒーローのみ使用)
) => number;    // 明度 0-1 (0=背景, 1=最も濃い文字)
```

### 共通ルール

- 返り値が0のセルには文字を描画しない（背景が見える）
- 返り値が0.01〜0.3: 疎な文字、低アルファ（形のエッジ部分）
- 返り値が0.3〜0.7: 中密度の文字（形の中間部）
- 返り値が0.7〜1.0: 密な文字、高アルファ（形の中心・暗部）
- 形の外側はすべて0を返すこと（白背景を活かすため）

### 統一文字セット

```
" .,:;+*?%S#@"
```

暗→明の12段階。空白は描画しない。

### 動きの原則

- すべてのモチーフはゆっくりと動き続ける
- 動きの速度: 人間が「静止画だと思ったら動いていた」と気づく程度
- 急な動きは禁止。最も速い動きでも3秒以上の周期
- ループは滑らかに。sin/cosの組み合わせで継ぎ目のない循環

## 11個のモチーフ詳細

### 1. 原爆ドーム（dome.ts）— ヒーロー用

**形状**: 正面から見た原爆ドームのシルエット。中央の楕円ドーム、その下の矩形建物、左右の壁面残骸。鉄骨の骨格が見える。

**構成要素**:
- ドーム: 上部の楕円。鉄骨のリブを放射状の線で表現
- 建物本体: ドーム下の矩形。窓の位置に暗部を配置
- 壁面: 左右に崩れかけた壁。不規則な上端ライン
- 地面の反射: 建物下に薄い反転影

**動き**: 
- ドーム内の鉄骨リブが微かに揺れる（風に見える）
- 全体にわずかな陽炎のような歪み
- 速度: sin波で周期6-8秒

**マウス追従**: マウス位置から光が当たっているように陰影が微かに変化

---

### 2. 折り鶴（crane.ts）

**形状**: 折り鶴を斜め上から見たシルエット。翼を広げた状態。

**構成要素**:
- 頭: 細い三角形、首から前方に伸びる
- 胴体: 中央のダイヤモンド形
- 翼: 左右に大きく広がる三角形。折り目の線あり
- 尾: 後方に伸びる三角形

**動き**:
- 翼が上下にゆっくり羽ばたく（上下角度±8度程度）
- 胴体がわずかに上下する（翼の動きに連動）
- 全体がほんの少し左右に揺れる（吊り下げられている感じ）
- 速度: 周期4-5秒

---

### 3. 路面電車（tram.ts）

**形状**: 広島電鉄の車両を横から見たシルエット。箱型。

**構成要素**:
- 車体: 角丸の長方形。窓が等間隔に並ぶ
- パンタグラフ: 車体上部にひし形のパンタグラフ
- 車輪: 下部に2組の車輪
- 線路: 最下部に細い水平線

**動き**:
- 車体全体がゆっくり左から右へ移動（画面を横切る）し、右端に消えたら左端から再出現
- 車輪が回転（車輪内部の小さなスポークが回る）
- パンタグラフがわずかに上下する
- 窓の中に影が移動する
- 速度: 横移動は周期15-20秒（非常にゆっくり）

---

### 4. 厳島鳥居（torii.ts）

**形状**: 海に立つ大鳥居。水面に半分映った状態。

**構成要素**:
- 笠木: 上部の反った横木
- 島木: 笠木の下の直線横木
- 柱: 2本の太い主柱
- 貫: 柱の間の横木
- 水面: 鳥居の下半分に波打つ水面
- 反射: 水面下に鳥居の揺れた反映

**動き**:
- 水面が波打つ。反射が水面の動きに合わせて歪む
- 鳥居本体は静止（構造物なので動かない）
- 水面レベルがわずかに上下（潮の満ち引き、非常にゆっくり）
- 速度: 波は周期3-4秒、潮は周期20秒

---

### 5. 鹿（deer.ts）

**形状**: 宮島の鹿。横向きに立っている。角がある。

**構成要素**:
- 頭: 小さな楕円。耳が2つ突き出る
- 角: 頭から上に伸びる枝状の角（3-4又）
- 胴体: 大きな楕円
- 脚: 4本の細い線。前脚と後脚のペア
- 尾: 短い三角

**動き**:
- 耳がときどきピクッと動く（片耳ずつ、不規則）
- 頭がゆっくり上下する（草を食べているような動き）
- 尾がわずかに左右に振れる
- 脚は静止（立っている状態を維持）
- 胴体が呼吸に合わせてわずかに膨張収縮
- 速度: 頭の動きは周期5-7秒、耳は3-4秒でランダム

---

### 6. もみじ（maple.ts）

**形状**: もみじの葉1枚。5つの鋭い裂片を持つ。

**構成要素**:
- 5つの裂片: 中央が最も大きく、左右に小さくなる
- 葉脈: 中央から各裂片に向かって伸びる線
- 葉柄: 下部から伸びる細い茎

**動き**:
- 葉全体がゆっくり回転する（1回転に30秒以上）
- 風に吹かれるように不規則な揺れが加わる
- 各裂片の先端が微かに波打つ
- 速度: 回転は極めて遅く、揺れは周期4-6秒

---

### 7. 牡蠣（oyster.ts）

**形状**: 牡蠣の殻。不規則な楕円形。表面にひだがある。

**構成要素**:
- 殻の輪郭: 不規則な楕円。上端はギザギザ
- ひだ: 殻表面に同心円状の成長線
- 内側: 殻が少し開いて中が見える（明るい部分）
- 殻の合わせ目: 中央付近に暗い線

**動き**:
- 殻がゆっくり開閉する（開いて少し見えて、また閉じる）
- 表面のひだが呼吸するように微かに動く
- 速度: 開閉は周期8-10秒

---

### 8. 橋（bridge.ts）

**形状**: アーチ橋を横から見た図。太田川に架かる橋。

**構成要素**:
- アーチ: 半円形のアーチ構造
- 路面: アーチの上を走る水平線
- 欄干: 路面の上に低い柵
- 橋脚: アーチの両端と中央の支柱
- 水面: 橋の下に水面

**動き**:
- 水面がゆっくり流れる（左から右へ）
- 水面に映る橋の反射が揺れる
- 橋自体は静止
- 速度: 水流は周期10-15秒

---

### 9. 平和の灯（flame.ts）

**形状**: 平和記念公園の「平和の灯」。台座の上で燃える炎。

**構成要素**:
- 炎: 中央の炎。先端が尖り、根元が広がる
- 炎の芯: 炎の中心にある最も明るい部分
- 台座: 炎の下の水平な台
- 光の拡散: 炎の周囲に薄く広がる光

**動き**:
- 炎が揺れる。先端の形が絶えず変化する
- 炎の芯がゆらゆらと左右に揺れる
- 光の拡散が炎の動きに合わせて脈動する
- これが11個の中で最も有機的な動きをするモチーフ
- 速度: 揺れは周期2-3秒（他より少し速い。炎なので）

---

### 10. お好み焼きのヘラ（spatula.ts）

**形状**: 広島お好み焼きで使う2枚のヘラ（コテ）。交差して置かれている。

**構成要素**:
- ヘラ1: 台形の金属部分 + 木の柄
- ヘラ2: ヘラ1と交差する形で配置
- 湯気: ヘラの上に立ち昇る湯気の線

**動き**:
- 湯気がゆっくり昇っていく（下から上へ、ゆらゆら）
- ヘラ自体は静止（鉄板の上に置かれている状態）
- 湯気の形が時間とともに変化
- 速度: 湯気の上昇は周期5-7秒

---

### 11. フェリー（ferry.ts）

**形状**: 宮島行きのフェリーを横から見たシルエット。

**構成要素**:
- 船体: 下部が丸みを帯びた台形
- 船室: 船体の上の箱型構造。窓が並ぶ
- 煙突: 船室の上に短い煙突
- 水面: 船の下に水面。船首に波が立つ
- 航跡: 船の後方に広がるV字の波

**動き**:
- 船体全体がゆっくり左右に揺れる（海の波で揺れる感じ）
- 船首の波が動く
- 航跡の波が後方に広がっていく
- 煙突から薄い煙が出る（湯気と同様の表現）
- 速度: 揺れは周期4-5秒

## 実装上の注意

### 形の定義方法

各モチーフは以下のアプローチで形を定義する:

1. **基本形状を数学関数で定義**: 楕円、矩形、三角形、円弧などの組み合わせ
2. **SDF（Signed Distance Function）を活用**: 形状の内外判定と滑らかなエッジ処理に最適
3. **パーツの重ね合わせ**: max()で合成。引き算はmax(a, -b)で
4. **陰影**: 形状内部で法線方向や中心からの距離に応じて明度を変化させる

### SDFの基本パーツ（実装時の参考）

```typescript
// 円
function sdCircle(px: number, py: number, cx: number, cy: number, r: number): number {
  return Math.sqrt((px-cx)**2 + (py-cy)**2) - r;
}

// 矩形
function sdBox(px: number, py: number, cx: number, cy: number, hw: number, hh: number): number {
  const dx = Math.abs(px - cx) - hw;
  const dy = Math.abs(py - cy) - hh;
  return Math.sqrt(Math.max(dx,0)**2 + Math.max(dy,0)**2) + Math.min(Math.max(dx,dy), 0);
}

// 楕円（近似）
function sdEllipse(px: number, py: number, cx: number, cy: number, rx: number, ry: number): number {
  const nx = (px - cx) / rx;
  const ny = (py - cy) / ry;
  return (Math.sqrt(nx*nx + ny*ny) - 1) * Math.min(rx, ry);
}

// 三角形
function sdTriangle(px: number, py: number, ax: number, ay: number, bx: number, by: number, cx: number, cy: number): number {
  // 実装は省略するが、3辺のSDF合成で作る
}

// 合成
function opUnion(d1: number, d2: number): number { return Math.min(d1, d2); }
function opSubtract(d1: number, d2: number): number { return Math.max(d1, -d2); }
function opSmooth(d1: number, d2: number, k: number): number {
  const h = Math.max(k - Math.abs(d1-d2), 0) / k;
  return Math.min(d1, d2) - h*h*k*0.25;
}
```

### 明度への変換

```typescript
// SDFの距離値を明度(0-1)に変換
function sdfToBrightness(d: number, edgeSoftness: number = 0.02): number {
  if (d > edgeSoftness) return 0; // 形の外
  if (d > 0) return (1 - d / edgeSoftness) * 0.3; // エッジ付近（薄い）
  // 形の内部: 中心ほど濃い
  return 0.3 + Math.min(1, Math.abs(d) * 5) * 0.7;
}
```
